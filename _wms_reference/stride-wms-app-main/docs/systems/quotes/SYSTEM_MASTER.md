## 1. FUNCTIONAL SCOPE

### DR-1 Functional scope
- **Primary purpose:**
  - `quotes` supports warehouse service quotations with class lines, selected services, rate overrides, status progression (`draft/sent/accepted/declined/expired/void`), magic-link acceptance, and quote event logging. **Confirmed**
    - Evidence: `public.quotes`, `public.quote_class_lines`, `public.quote_selected_services`, `public.quote_rate_overrides`, `public.quote_events` definitions and enums. (`supabase/migrations/20260127210000_quoting_tool.sql`: `CREATE TABLE public.quotes`, `CREATE TYPE quote_status`, `CREATE TYPE quote_event_type`).
  - `repair_quotes` supports technician and client workflow (`draft` through `closed`), tokenized technician/client links, item-level allocations, and audit trail JSON. **Confirmed**
    - Evidence: `repair_quote_workflow_status`, `repair_quote_items`, `repair_quote_tokens`, `repair_quotes.status`, `repair_quotes.audit_log`. (`supabase/migrations/20260125045116_update_repair_quotes_schema.sql`).

- **Explicit non-goals (from code evidence):**
  - No integrated quote-to-invoice conversion implementation in these quote hooks/pages; billing internals are external to this slice. **Confirmed**
    - Evidence: `useQuotes`/`QuoteAcceptance` update quote + quote_events only; no billing write in those flows. (`src/hooks/useQuotes.ts`, `src/pages/QuoteAcceptance.tsx`).

- **Cross-system touchpoints:**
  - **Tasks:** repair quote can be seeded from task (`source_task_id`) and accepted repair quote can auto-create a repair task via DB trigger. **Confirmed**
  - **Billing:** quote/repair quote flows here do not directly insert billing events. **Confirmed**
  - **Claims:** claim items can link to repair quote IDs in claim UI. **Confirmed**
  - **Client Portal:** client-facing repair quote list + accept/decline UI + RLS policies. **Confirmed**
  - **Alerts/Email:** send-to-client and send-to-tech create tokens and queue alert emails. **Confirmed**
  - **Inventory:** no direct item status mutation found in quote acceptance paths. **Risk** (possible missing intended side effects).

## 5. SEQUENTIAL DATA FLOW

### Flow: Service Quote Send + Accept
1. Staff sends quote (`useQuotes.sendQuote`) -> quote set to `sent` with new `magic_link_token`.
2. Email sent with accept URL.
3. Client opens `QuoteAcceptance` with token.
4. Client accepts/declines -> update `quotes.status` + timestamp.
5. `quote_events` row inserted.

### Flow: Repair Quote Lifecycle
1. Staff creates workflow quote (`createWorkflowQuote`) optionally from task (`source_task_id`).
2. Staff sends to technician (`sendToTechnician`) -> token + `sent_to_tech`.
3. Technician submits (`useTechQuoteSubmission.submitQuote`) -> `tech_submitted` with totals.
4. Staff reviews/prices -> optionally `under_review`, updates office pricing.
5. Staff sends to client (`sendToClient`) -> token + `sent_to_client`.
6. Client accepts (`useClientQuoteReview.acceptQuote`) -> `accepted` + response fields.
7. DB trigger `create_repair_task_on_quote_acceptance` creates repair task + `task_items`; sets `repair_task_id`.

## 8. STAFF USAGE GUIDE (STEP-BY-STEP)

### How to create and send a repair quote
1. Open a task with repairable items.
2. Use “Create Quote” from task detail.
3. Assign technician and send technician link.
4. After technician submission, review and set customer price.
5. Send to client; monitor accepted/declined status.

### How to send a service quote
1. Build quote in quote builder.
2. Send quote to customer email.
3. Customer accepts/declines via magic-link page.

## 9. MARKETING WRITE-UP

Stride WMS provides two quote systems: service quotes for warehouse pricing and repair quotes for item-specific repair approvals. Teams can generate, send, and track quotes with clear status history and customer response capture.

Repair quote workflows support technician collaboration, office pricing control, and client decisioning. Upon acceptance, repair task creation can be automatically linked, improving operational continuity.
