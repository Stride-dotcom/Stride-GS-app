#### Decision
When a tenant admin adds/removes staff users, the system must track the tenant’s billable staff seat count and automatically sync Stripe subscription billing by setting the per-user subscription item quantity to that count.

#### Why
Seat-based billing must remain accurate without manual Stripe operations; billing changes should follow staff onboarding/offboarding so Stripe invoices reflect the correct seat quantity.

#### Implementation impact
- Add DB logic to compute billable seat count deterministically.
- Add an Edge Function to apply seat quantity updates to the tenant’s Stripe subscription.
- Wire seat-sync calls into staff management flows and optionally expose a manual “sync seats” action in Billing.
